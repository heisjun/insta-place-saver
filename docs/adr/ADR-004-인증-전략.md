# ADR-004: 인증 전략

## 배경

개인용 앱이므로 회원가입 폼 없이 소셜 로그인만 제공한다.
Supabase Auth를 인증 레이어로 사용한다.

## 결정

**카카오 + 구글 OAuth** 제공, Supabase Auth 연동

## 구현 상세

**서버/클라이언트 분리:**
- 서버: `src/lib/supabase/server.ts` — `@supabase/ssr` 기반, cookies() 사용
- 클라이언트: `src/lib/supabase/client.ts` — `createBrowserClient`
- 미들웨어: `src/proxy.ts` — 세션 갱신 처리

**보안:**
- `supabase.auth.getSession()` 대신 `supabase.auth.getUser()` 사용
  - `getSession()`은 쿠키 값을 그대로 반환 (서버 검증 없음)
  - `getUser()`는 Supabase Auth 서버에 실제로 검증 요청
- RLS(Row Level Security)로 `auth.uid() = user_id` 강제

**Next.js 16 변경사항:**
- `src/middleware.ts` → `src/proxy.ts` 로 파일명 변경 (Next.js 16 breaking change)
- export 함수명도 `middleware` → `proxy` 로 변경

## 카카오 로그인 설정 시 주의사항

카카오 개발자 콘솔에서 두 곳에 도메인을 따로 등록해야 한다:
1. **플랫폼 → Web → 사이트 도메인**: 일반 요청 허용
2. **플랫폼 키 → JavaScript 키 → JavaScript SDK 도메인**: 카카오맵 JS SDK 사용 허용

두 곳이 다름. Kakao Maps 401 에러는 2번이 누락된 경우 발생.
