# ADR-002: 인스타그램 캡션 추출 전략

## 배경

사용자가 인스타그램 게시글 URL을 입력하면 캡션을 자동으로 가져와야 한다.
공식 API는 앱 심사가 필요해 개인 프로젝트에서 현실적으로 불가능하다.

## 시도한 방법 (모두 실패)

자세한 분석은 `docs/instagram-scraping-research.md` 참고.

1. **Instagram 비공식 GraphQL API** (`doc_id` 방식) → HTML 반환 (봇 감지)
2. **Instagram Embed 페이지 파싱** → JS 렌더링 필요, 정적 HTML에 데이터 없음
3. **Instagram oEmbed API** → `content-length: 0` (인증 필요)
4. **`?__a=1` JSON API** → 404 (엔드포인트 폐지)
5. **미러 사이트 (Picuki, Imginn)** → Cloudflare Bot Management 차단
6. **got-scraping (TLS 핑거프린트 우회)** → IP 차단으로 여전히 403
7. **Selenium/Puppeteer** → Vercel 서버리스 환경에서 운용 복잡, Cold Start 문제

## 결정

**Apify Instagram Scraper** (Actor ID: `shu8hvrXbJbY3Eb9W`) 사용

```
우리 앱 → POST Apify API → Apify 서버(크롤링) → JSON 결과 반환
```

Apify가 자체 인프라에서 크롤링을 처리하므로 TLS 핑거프린팅, IP 차단, Cloudflare 문제를 우리가 직접 해결할 필요 없다.

**폴백**: Apify 실패 시 수동 캡션 붙여넣기 UI 노출 (`/add` 페이지 에러 상태)

## 근거

- 서버리스(Vercel) 환경에서 Puppeteer 운용이 복잡하고 Cold Start 문제 존재
- Apify 무료 플랜 $5/월 크레딧으로 개인 사용량(월 100회 이하) 감당 가능
- 크롤링 유지보수 부담을 Apify 측에 위임

## 제약

- Apify 응답 시간 30~60초 (브라우저 실행 구조)
- 비공개 계정 게시글 접근 불가
- Apify 정책/요금 변경 시 영향받음

## 환경변수

```
APIFY_TOKEN=apify_api_xxx  # apify.com → Settings → API tokens
```
